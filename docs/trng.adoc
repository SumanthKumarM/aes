== Overview

This document describes the hardware *True Random Number Generator (TRNG)* used to generate *fresh randomness `R`* required for masked arithmetic in the masked AES S-Box.

The generated random values serve as the *fresh mask input `R`* in every masked multiplication operation.

== Architecture Overview

The TRNG follows a classical entropy-conditioning pipeline:

----
Physical Noise (CMOS thermal/flicker jitter)
            │
            ▼
  Ring Oscillator Array (32 × 13-stage)
            │
            ▼
       XOR Mixing Tree
            │
            ▼
     D Flip-Flop Sampler
            │
            ▼
       Health Tests
       ┌────┴────┐
      RCT       APT
       └────┬────┘
            │
            ▼
  Entropy Collector (256-bit SIPO)
            │
            ▼
  Keccak-f[1600] Conditioning (24 rounds)
            │
            ▼
   Output Formatter (4-bit words)
            │
            ▼
       R₁ … R₇ per S-box
----

== Entropy Source — Ring Oscillator Array

Randomness originates from physical noise in CMOS circuits — thermal noise, flicker noise, supply voltage variation, temperature fluctuation, and transistor delay variation. Together these introduce *timing jitter* in logic gate propagation delays.

=== Ring Oscillator Structure

Each oscillator is a chain of an odd number of inverters, which is what sustains continuous oscillation:

----
INV → INV → INV → ... → INV
----

This design uses 32 ring oscillators, each 13 inverters deep.

== Entropy Amplification via XOR Mixing

Each ring oscillator's output carries timing jitter. To raise entropy quality, all 32 outputs are combined through an XOR tree:

----
RO₀  ─────┐
RO₁  ─────┤
RO₂  ─────┤
 :        ⊕ ───▶ Output
 :        │
RO₃₁ ─────┘
----

XOR mixing pools multiple independent noise sources into one composite signal with higher aggregate entropy than any single oscillator.

== Sampling the Entropy Signal

The XOR tree output is sampled by a D flip-flop clocked independently of the oscillators:

----
RO XOR output → DFF → raw_bit_stream
----

Sampling turns timing jitter into a stream of digital random bits, one bit per clock cycle.

== Health Tests

The raw entropy stream is continuously monitored using runtime health tests to verify that the entropy source has not failed or become biased.

*Runtime Error Handling:* If either test fails, the TRNG immediately transitions to an unrecoverable DEAD state, halting all key generation. The device requires an external reset (`ext_rst_n`) to resume operation. No error recovery is attempted; any detected anomaly is treated as a critical failure.

=== Repetition Count Test (RCT)

* *Purpose:* Detects if the entropy source becomes "stuck."
* *Mechanism:* Monitors and counts consecutive identical bits in the stream.
* *Failure Condition:* If the repetition count exceeds a predefined threshold, an error is raised.
* *Example Failure:* `111111111111111111...`
* *Action on Failure:* TRNG enters DEAD state → waits for external reset.

=== Adaptive Proportion Test (APT)

* *Purpose:* Detects bias in the bitstream within a specific data window.
* *Window Size:* *1024 bits* (per *NIST* standard).
* *Mechanism:* Operates within a sliding window to check the statistical distribution of bits.
* *Failure Condition:* If the number of `1` bits exceeds a calculated statistical threshold, the entropy source is considered faulty.
* *Action on Failure:* TRNG enters DEAD state → waits for external reset.

== Control Unit FSM

The Control Unit manages the TRNG state machine using a fail-fast error handling model:

* *IDLE:* Initialization state; noise source and health tests are disabled; Keccak held in reset.
* *BIST:* Built-in Self-Test state; health tests continuously monitor the raw bitstream; Keccak processes entropy.
  - If `health_error` is asserted, immediate transition to DEAD state occurs (no recovery mechanism).
* *WAIT_FOR_XFER:* Awaiting handshake from S-Box; key output is available.
  - If `health_error` is asserted, immediate transition to DEAD state occurs.
* *DEAD:* Unrecoverable failure state; all outputs disabled; key generation is halted.
  - Remains in DEAD state until external reset (`ext_rst_n`) is applied.

*Design Rationale:* Any detected error is treated as fatal. This approach simplifies the implementation and ensures that anomalies immediately prevent further random generation, protecting against side-channel leakage from a compromised entropy source.

== Entropy Collector

The entropy source produces *one random bit per clock cycle*.
Bits are collected using a *Serial-In Parallel-Out (SIPO) shift register*:

----
raw bits: 1 0 1 1 0 0 1 0

shift register output:
1011
0010
----

Configuration used in this design:

----
entropy_pool_size = 256 bits
----

These bits are injected into the Keccak conditioning block.

== Keccak Conditioning

The TRNG uses the *Keccak permutation* as a cryptographic conditioning function — the same algorithm used in *SHA-3*.

Conditioning removes bias, destroys correlation between samples, and diffuses entropy across the full state.

=== Keccak State

The permutation operates on a *1600-bit internal state*:

----
5 × 5 matrix of 64-bit lanes
Total: 5 × 5 × 64 = 1600 bits
----

=== Absorb Phase

Entropy bits are injected into the state via XOR:

----
state = state XOR entropy_block
----

Incoming entropy mixes with the existing state rather than overwriting it.

=== Keccak Permutation — Keccak-f[1600]

The permutation executes *24 rounds*, each applying:

----
θ → ρ → π → χ → ι
----

==== Theta (θ) — Column Parity Mixing

----
C[x] = A[x,0] ⊕ A[x,1] ⊕ A[x,2] ⊕ A[x,3] ⊕ A[x,4]
D[x] = C[x−1] ⊕ ROT(C[x+1],1)
A[x,y] = A[x,y] ⊕ D[x]
----

==== Rho (ρ) — Lane Rotation

----
A[x,y] = ROT(A[x,y], offset[x,y])
----

Rotation offsets are fixed constants defined in the Keccak specification.

==== Pi (π) — Lane Permutation

----
A'[x,y] = A[(x + 3y) mod 5][x]
----

This spreads information throughout the state.

==== Chi (χ) — Nonlinear Step

----
A[x,y] = A[x,y] ⊕ ((¬A[x+1,y]) ∧ A[x+2,y])
----

==== Iota (ι) — Round Constant Injection

----
A[0,0] = A[0,0] ⊕ RC
----

Each round uses a unique constant.

=== Squeeze Phase

After the permutation completes, the Keccak state contains *high-quality pseudorandom bits*:

----
state → random bit stream
----

== Random Output Formatting

The conditioned output stream is formatted into *4-bit random values*:

----
random stream: 101001110010...

4-bit outputs:
1010
0111
0010
...
----

These values serve as the *fresh randomness `R`* used in masked multiplication.

== Integration with Masked AES

The TRNG supplies randomness to masked arithmetic units.

Each masked multiplication consumes one fresh R value:

----
C₁ = A₁B₁ ⊕ R
C₂ = A₁B₂ ⊕ A₂B₁ ⊕ A₂B₂ ⊕ R
----

Where `R = TRNG output`. Each multiplication must use *fresh* randomness.

The S-Box implementation requires *7 random values per operation* (R₁ … R₇).

== Security Properties

The design combines a physical entropy source (ring-oscillator jitter), entropy amplification through XOR mixing, runtime health monitoring (RCT and APT), and cryptographic conditioning via Keccak-f[1600] to continuously generate fresh randomness resistant to first-order side-channel analysis.

== Verification

TRNG verification is performed using Cocotb-based simulation:

* Entropy injection models
* Bias detection tests
* Stuck-bit fault scenarios
* Health test validation
* Keccak permutation verification

== Summary

The TRNG turns physical hardware noise into cryptographically strong randomness through the conditioning pipeline described above, supplying the R₁ … R₇ values that protect masked AES operations against side-channel attacks.
