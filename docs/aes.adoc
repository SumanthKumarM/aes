== ShiftRows & MixColumns

The `shiftRows` and `mixColumns` modules are pure combinational datapath blocks without independent design derivation. Like the `cipher` module, they are direct structural implementations of the FIPS 197 `ShiftRows()` (Section 5.1.2) and `MixColumns()` (Section 5.1.3) procedures, operating on the `state_matrix_t` type used throughout the datapath.

`shiftRows` implements the per-row cyclic left shift in a single `always_comb` loop over all 16 state bytes:

----
s'[r][c] = s[r][(c + r) mod 4]      for 0 <= r, c < 4
----

Row 0 is unchanged; rows 1–3 are cyclically shifted left by 1, 2, and 3 byte positions respectively.

`mixColumns` implements the fixed GF(2^8) matrix multiplication of Eq. 5.7/5.8, applied independently to each of the four columns via a shared `column_op()` function:

----
[02 03 01 01]   [s0,c]
[01 02 03 01] x [s1,c]
[01 01 02 03]   [s2,c]
[03 01 01 02]   [s3,c]
----

`cipher` instantiates both blocks directly (see below) and sequences them between `SubBytes` and `AddRoundKey` each round, per FIPS 197 `CIPHER()`.

== Cipher

The `cipher` module is the top-level datapath that coordinates `SubBytes` (via `sbox`), `ShiftRows`, `MixColumns`, and `AddRoundKey` into the complete AES encryption round structure. Unlike the S-Box, InvSBox, TRNG, and AddRoundKey blocks, the `cipher` module contains no independent design derivation. It is a direct structural implementation of the FIPS 197 `Cipher()` procedure (Section 5.1):

----
procedure CIPHER(in, Nr, w)
    state <- in
    state <- ADDROUNDKEY(state, w[0..3])
    for round from 1 to Nr-1 do
        state <- SUBBYTES(state)
        state <- SHIFTROWS(state)
        state <- MIXCOLUMNS(state)
        state <- ADDROUNDKEY(state, w[4*round..4*round+3])
    end for
    state <- SUBBYTES(state)
    state <- SHIFTROWS(state)
    state <- ADDROUNDKEY(state, w[4*Nr..4*Nr+3])
    return state
end procedure
----

The `cipher` module instantiates `sbox`, `shiftRows`, `mixColumns`, and `addRoundKey` as sub-modules, controlling their sequencing through an internal FSM (`PRE_ADDROUNDKEY` / `ADDROUNDKEY` states) that is gated by `round_cntr` and `Nr` (the round count for the active key size — 10/12/14 for AES-128/192/256 per link:keyExp_logic.adoc[KeyExpansion & AddRoundKey]). The routing of random bits from the TRNG to the S-Box and the S-Box/AddRoundKey handshake timing follow the conventions established in link:sbox.adoc[Composite Masked AES S-Box Implementation] and link:trng.adoc[TRNG Implementation].
